# Reports

All reports are plain Markdown files written to `$REPORTS_DIR`. No database, no proprietary format.

---

## Directory Structure

```
$REPORTS_DIR/
  index.md                        ← master index, last 7 days
  baseline/
    ports.json                    ← known open ports baseline
  2026-03-14/
    10-23-incident-reforms-django.md
    14-05-security-scan.md
    daily-summary.md
  2026-03-13/
    ...
```

---

## File Naming

### Incident reports
```
HH-MM-incident-<container>.md
```
Example: `10-23-incident-reforms-django.md`

### Security scan reports
```
HH-MM-security-scan.md
```
Example: `14-05-security-scan.md`

### Daily summary
```
daily-summary.md
```
One per day, overwritten on each `report --type daily` run.

## Incident Report

Written by `write_incident_report()` whenever a container has anomalies.

```markdown
# Incident: reforms-django
**Time:** 2026-03-14 10:23 UTC
**Severity:** 🔴 CRITICAL
**Status:** RESOLVED

## Container State
- **Image:** reforms:latest
- **Status:** Up 5 minutes (unhealthy)
- **State:** unhealthy
- **Restart count:** 1

## Anomalies
| Severity | Type | Detail |
|----------|------|--------|
| 🔴 CRITICAL | unhealthy | Health check failing |

## Actions Taken
| Action | Command | Result |
|--------|---------|--------|
| restart | docker restart reforms-django | Container restarted |
```

Status is `RESOLVED` if healing succeeded, `ESCALATED` if not.

Written by `write_security_report()` after a `scan` run.

```markdown
|----------|------|--------|
| 🟡 WARNING | 9090 | Port not in baseline |
```

## Index

`index.md` is regenerated by `update_index()` after every report write. It lists all report files from the last 7 days.

```markdown
# Server Agent Reports

## 2026-03-14
- [10-23-incident-reforms-django.md](2026-03-14/10-23-incident-reforms-django.md)
```

## Port Baseline

`baseline/ports.json` is generated by `server-agent scan --baseline` and used by subsequent scans to detect unexpected ports.

```json
{
  "reviewed": false,
  "generated_at": "2026-03-14T10:00:00Z",
  "ports": {
    "80": {"service": "traefik", "expected": true},
    "443": {"service": "traefik", "expected": true},
    "7000": {"service": "carapis-django", "expected": true}
  }
}
```

After generating: review the file, set `"reviewed": true`, commit it to version control.

---

## Environment Variable

```bash
export REPORTS_DIR=/root/server-agent/reports
```

Required. All report writing fails if not set. Set in `/etc/environment` or the systemd unit file for the cmdop agent.
